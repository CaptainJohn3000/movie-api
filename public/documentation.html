Below you will find my documentation for my Movie-flix API.

Layout

Request: URL: HTTP Method: Request data format: Response data format:

Getting Home Page with Welcome:

/ GET None Will receive "Welcome!"

Get a list of movies:

/movies GET None Json object with data about all movies in API

Return data about a single movie by title to user:

/movies/:title GET Insert movie title you want to request in URL [HERE] Json object with data about a single movie, containing description, genre, director, and image URL.

Return data about a genre by type:

/genre/:type GET Insert movie title you want to request in URL [HERE] JSON object with data about a genre by Name

Return data about a director by name:

/directors/:director GET Insert directors name in URL [HERE] JSON object with data about specific director that was requested

Allow new users to register:

/users/register POST JSON object with data about the new user Example: "userName" : "API1234" "firstName" : "Tanner" "lastName" : "Free" "password" : "12345" "email" : "email@emailaddress.com" "birthDate" : "DOB here" "id" : "ID HERE" If successful, you will receive a JSON object with the data of the new user, and new user's ID.

Allows users to update their username:

/users/:id PUT JSON oject with the data to be updated. Example: { "Username" : "API5544" } JSON oject with the user's data, including the updated username, showing that it has been successful.

Allow users to add/remove a movie to their favorites, showing only a text that the movie has been added:

users/:id/favorites/:title POST Insert which user ID you want to add a movie to your favorites. [MOVIE ID] is to be the ID of the movie to be added to favorites. A message will be returned indicating the movie has been added successfully to the specific user's favorites list.

Allow existing users to deregister:

/users/:id/deregister DELETE Insert which user ID in [USER ID], that you wish to deregister. A message confirming success or failure will be sent.
